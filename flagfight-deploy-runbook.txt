How deployment is structured
- The CloudPanel htdocs folder is NOT the Git working folder.
- Do not run git pull inside /home/flagfight/htdocs/flagfight.bidnsteal.com.
- Safe deployment flow:
  1. Pull latest code in /home/flagfight/repo/flag-fight
  2. Install deps and build in the Git clone
  3. rsync built repo into CloudPanel htdocs
  4. Install production deps in htdocs
  5. Restart flagfight.service
  6. Check systemd status

One-command deployment from this PC (run after git push)

$script = @'
set -euo pipefail
export PATH=/home/flagfight/.nvm/versions/node/v22.22.2/bin:/usr/local/sbin:/usr/local/bin:/usr/sbin:/usr/bin:/sbin:/bin
repo=/home/flagfight/repo/flag-fight
app=/home/flagfight/htdocs/flagfight.bidnsteal.com
cd "$repo"
git pull --ff-only origin main
git rev-parse --short HEAD
npm ci
npm run build
rsync -a --delete --exclude '.git/' --exclude 'node_modules/' --exclude '.env' "$repo"/ "$app"/
chown -R flagfight:flagfight "$app"
cd "$app"
npm ci --omit=dev
systemctl restart flagfight.service
sleep 5
systemctl is-active flagfight.service
'@
$tmp = New-TemporaryFile
Set-Content -LiteralPath $tmp -Value $script -Encoding ASCII
scp -i "C:\Users\ashar\Documents\sshkey.key" $tmp ubuntu@168.138.115.223:/tmp/flagfight_deploy.sh
ssh -i "C:\Users\ashar\Documents\sshkey.key" ubuntu@168.138.115.223 "sudo -n bash /tmp/flagfight_deploy.sh"
Remove-Item -LiteralPath $tmp
